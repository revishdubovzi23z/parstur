# TMDB Folders for Lampa

Плагин для [Lampa](https://lampa.mx/) (и совместимых форков, например Al(co)pac
с `wiki.alcopa.cc`), который добавляет в левое меню пункт **«TMDB папки»**.
При его выборе сначала показывается список твоих **списков (folders)** на TMDB,
а при выборе папки — её содержимое (фильмы / сериалы). Карточки внутри папок
ведут себя как обычные TMDB-карточки Lampa — открываются стандартным экраном
просмотра с балансерами / онлайн-источниками.

## Быстрый старт

1. Залей файл `tmdb_folders.js` на любой HTTPS-хост, доступный твоему Lampa
   (например, в `static/` своего VPS, на GitHub Pages, в админ-панель Al(co)pac).
2. В Lampa зайди в **Настройки → Расширения → Добавить плагин** и укажи URL
   к этому JS-файлу. (В Al(co)pac можно загрузить плагин через админ-панель —
   результат тот же.)
3. Открой **Настройки → TMDB папки** и заполни:
   - **TMDB v4 Bearer token** *(обязательно)* — Read Access Token из
     `https://www.themoviedb.org/settings/api` (длинная строка вида `eyJ…`).
   - **TMDB account\_object\_id** *(опционально)* — нужен, чтобы плагин сам
     подтянул все твои списки. ID можно посмотреть, например, через запрос
     `GET /3/account` или `GET /4/account/{username}`. Без него плагин
     обойдётся, если задать ID списков вручную.
   - **Список ID папок (через запятую)** *(опционально)* — `123, 456, 789`.
     Удобно для публичных списков и/или если `account_object_id` не задан.
   - **Язык TMDB (BCP-47)** *(опционально)* — например `ru-RU`. По умолчанию
     берётся из текущего языка Lampa.
   - **TTL кэша** — как долго плагин хранит ответы TMDB в памяти, чтобы не
     перегружать API.
4. Перезайди в Lampa: в левом меню появится пункт **TMDB папки**.

## Как это работает

- Все запросы идут на `https://api.themoviedb.org` с заголовком
  `Authorization: Bearer <token>`.
- Плагин держит **очередь с дросселированием ≥200 мс** между запросами, чтобы
  оставаться существенно ниже лимита TMDB (~40 запросов / 10 секунд).
- Ответы кэшируются в памяти на время TTL (по умолчанию 10 минут).
- Папки берутся через `GET /4/account/{account_object_id}/lists`
  (постранично). Если этот эндпоинт недоступен (например, при использовании
  только read-токена без user-токена), плагин автоматически переходит на
  ручной список ID, заданный в настройках.
- Содержимое папки получается через `GET /4/list/{list_id}?page=N&language=…`.

## Совместимость

- Lampa официальная сборка (yumata/lampa-source), `app_digital >= 242`.
- Форки: Al(co)pac, Lampa UNCENSORED, bylampa-сборки и т. п. — должны работать,
  так как используется только публичный `Lampa.*` API.

## Что НЕ делает плагин

- Не пишет в TMDB (плагин read-only).
- Не модифицирует балансеры и плееры Lampa.
- Не читает файловую систему сервера — никакого бекенда на VPS не требуется.

## Известные ограничения

- Для **приватных** TMDB-списков нужен v4 **user access token** (не application
  read-token). Если в Lampa-настройках указан только read-token, увидеть
  получится только публичные списки.
- TMDB не возвращает `media_type` для всех элементов списков; плагин
  определяет тип по наличию `first_air_date` → `tv`, иначе `movie`.
