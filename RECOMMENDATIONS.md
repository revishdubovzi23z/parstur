## ОСТАЁТСЯ ОТКРЫТЫМ (12 из 24)

### АРХИТЕКТУРНЫЕ ПРОБЛЕМЫ

#### 4. Нет единого слоя доступа к БД
**Проблема:** Каждый файл создаёт своё подключение к SQLite: `main.py` → `get_db()`, `rezka_sync.py` → свой `get_db()`, `sync_job.py` → свой `TrackerAppCore`, и т.д. Нет общей ORM или DAL.  
**Решение:** Создать единый `db.py` с `get_db()`, общими методами `get_items()`, `update_item()`, `insert_release()`, `get_feed()`.

#### 5. `app_core.py.TrackerAppCore` — почти все методы-заглушки
**Проблема:** Stub-методы (`pass`, `return []`), реальная логика размазана по другим файлам. Локальный `TrackerAppCore` в `sync_job.py` затеняет класс из `app_core.py`.  
**Решение:** Перенести реальную логику в `TrackerAppCore` или удалить stub-класс.

#### 6. Нет миграций БД
**Проблема:** Схема меняется через `ALTER TABLE ADD COLUMN` в разных файлах. Нет отслеживания версий, нет отката.  
**Решение:** Добавить систему миграций (таблица `schema_version`, папка `migrations/`).

---

### НЕСОГЛАСОВАННОСТИ

#### 12. Несогласованность полей `rating` между API-клиентами
**Проблема:** `kinopoisk_client`/`poiskkino_client` возвращают `kp_rating`/`imdb_rating`, а `tmdb_client` возвращает `rating` (vote_average). Разные шкалы, нет проверки что куда пишется.  
**Решение:** Явно разделить поля или добавить `rating_source`.

---

### ДУБЛИРОВАНИЕ КОДА

#### 13. Дублирование парсинга в `rutor_parser.py`
**Файл:** `rutor_parser.py:62–111` vs `137–181`  
**Проблема:** `get_category_releases()` и `search_releases()` содержат почти идентичный код парсинга HTML-таблицы.  
**Решение:** Вынести общий парсинг в `_parse_release_rows()`.

#### 14. Дублирование клиентов API
**Проблема:** `kinopoisk_client.py` и `poiskkino_client.py` имеют одинаковую структуру.  
**Решение:** Создать базовый класс `BaseMovieClient`.

#### 15. Дублирование логгера (перенаправление stdout)
**Проблема:** Класс `Logger` дублируется в `fix_posters.py` и `user_sync.py`. Глобальное перенаправление stdout — опасно.  
**Решение:** Создать единый `logger.py` на стандартном `logging`.

---

### ПРОИЗВОДИТЕЛЬНОСТЬ

#### 16. `cleanup_duplicates.py` грузит ВСЕ записи в память
**Решение:** Обрабатывать батчами через `fetchmany()`.

#### 17. PWA кеширует только 3 файла
**Решение:** Добавить cache-first для постеров, stale-while-revalidate для API.

#### 18. Нет пакетной обработки в `rezka_sync.py`
**Решение:** Использовать `asyncio` для параллельных запросов с ограничением concurrency.

---

### МЕЛКИЕ УЛУЧШЕНИЯ

#### 23. `index.html` — 1480+ строк в одном файле
**Решение:** Разбить на Vue SFC-компоненты, собрать через Vite.

#### 24. Нет тестов
**Решение:** Добавить `pytest` + тесты для `normalize_title()`, `clean_display_title()`, `parse_rutor_date()`, `deduplicate_releases()`.
